- **style:** dark, bento
- **palette:** bg `#0A0A0B` · ink `#EDEDED` · accent `#2F6BFF` — Reserved almost entirely for the product UI inside the screenshots — the selected-node highlight, the active 'Live' workflow pill, and small interactive cursors/handles. - **type:** display *Custom geometric grotesque (reads like a tightened Neue Montreal / GT America Mono-adjacent display cut; single-story 'a', near-circular 'o', minimal contrast)* · body *Neutral grotesque sans (Inter-like) at a muted grey weight* — Engineered, calm, infrastructural — display set huge and tight-leading so two lines stack into a slab; nothing decorative, lowercase-feeling even when capitalized
- **sections:** hero › logos › problem › feature-infrastructure › feature-capabilities › testimonials › how-it-works › feature-source-of-truth › feature-tooling › feature-prompts-to-systems › feature-audit-trail › feature-enterprise › cta › footer
- **signature:** The hero swaps the usual centered-headline + abstract render for an oversized left-aligned two-line statement up top and a giant, pixel-accurate live workflow-builder screenshot bleeding off the bottom edge — the product UI itself (nodes, side panels, a person-summary card with a real named contact) is the hero image, cropped so you feel you're looking over someone's shoulder mid-build.
- **imagery:** Product-screenshot as primary visual language, rendered dark-on-dark so the app dissolves into the page background. Heavy use of true UI chrome (node graphs, inspector panels, data tables) rather than illustrated abstractions. Logo row is fully desaturated to a uniform grey, fading at the edges. No 3D, no gradient mesh, no stock photography — credibility is carried by showing the actual tool.
- **copy:** Declarative, imperative-verb category-defining voice — leads with the action and the buyer's domain, not the tech. Hero headline (verbatim): 'Deploy agents that work across your go-to-market', subhead: 'Build, deploy, and improve your existing agents for every go-to-market function.'

**Takeaways (steal as ideas, don't copy):**
- Make the product the hero: crop a real, populated app screenshot (with believable sample data — a named person, real-looking emails, live status pills) and bleed it off the bottom of the fold instead of using an abstract render.
- Quarantine your accent color to the product UI only. Keep all marketing chrome monochrome so the single blue reads as 'this is the live app,' not decoration.
- Use a section-heading structure that names the buyer's stack, not your features: 'Your revenue stack wasn't built for agents' (problem) → 'The infrastructure agents need to run on your revenue stack' (positioning) frames you as a category, not a tool.
- Set the hero headline oversized with tight leading so two lines lock into a solid type-block, left-aligned, and pair it with a small muted-grey subhead off to the side rather than centered beneath — asymmetry signals 'engineering product, not marketing site.'
