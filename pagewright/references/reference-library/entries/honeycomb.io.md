It dramatizes human+agent shared debugging in the product's own voice rather than showing dashboards.
- **imagery:** No photography. A single dark mono-spaced terminal card floats on a soft cool-gray gradient, anchored by a pixel-art robot avatar. The aesthetic is illustration-light and code-native; the bee/honeycomb metaphor (hex logo, "Pollinating" copy, amber dot) carries brand warmth into an otherwise restrained light layout.
- **copy:** Two-tone color-split headline that leads with the category word and stakes the AI-era moment; technical yet plain-spoken. Hero: "Observability built for the AI era" (accent on "Observability").

**Takeaways (steal as ideas, don't copy):**
- Color-split the hero H1: render the category noun in accent blue and the rest in ink, so the page states its category and its angle in one glance.
- Replace the obligatory product-dashboard screenshot with a faux agent terminal that role-plays a real user question ('Our customers say checkout is slow right now') — show the workflow, not the UI chrome.
- Smuggle the brand metaphor into micro-copy: a loading spinner that reads 'Pollinating...' with an amber dot turns a system state into personality.
- Pair two CTAs as filled-vs-outline (Get a Demo / Start for Free) to serve enterprise and self-serve buyers without visual hierarchy conflict.
